文件必须来自作者指定下载地址；不要修改安装包文件名里的扩展名。

macOS 打开方式

优先使用系统设置里的"仍要打开"。如果遇到"文件已损坏"，再使用终端移除隔离属性。

Apple 官方说明

情况一：提示"无法验证开发者"

这是 macOS Gatekeeper 的正常保护。请先尝试下面这种系统推荐方式。

1

先双击打开一次应用，让 macOS 弹出拦截提示。

2

打开 系统设置 → 隐私与安全性。

3

在页面底部的"安全性"区域，找到刚刚被拦截的应用，点击 仍要打开 / Open Anyway。

4

输入电脑登录密码，再点击 打开。之后一般就可以正常启动。

"仍要打开"按钮通常只会在尝试打开后的短时间内出现。

情况二：提示"文件已损坏，无法打开"

如果你确定安装包来自可信渠道，可能是下载文件带有 quarantine 隔离属性。可以执行下面命令。

1

把应用拖到 应用程序 / Applications 文件夹。

2

打开 终端 Terminal。

3

复制下面命令执行，然后重新打开应用。

xattr -dr com.apple.quarantine "/Applications/Luna AI Cut.app"

Windows 打开方式

Windows SmartScreen 会根据应用签名和信誉判断风险。未签名测试版经常会被标记为"无法识别"。

Microsoft 安全说明

情况一：出现"Windows 已保护你的电脑"

确认文件来源可信后，可以手动展开更多选项。

1

在蓝色 SmartScreen 提示窗口中，点击 更多信息。

2

确认应用名称和发布渠道无误。

3

点击 仍要运行 / Run anyway。

4

如果弹出用户账户控制提示，点击 是 继续安装或启动。

情况二：文件属性里有"解除锁定"

某些下载文件会被 Windows 标记为来自互联网，右键属性里可以解除阻止。

1

右键安装包或应用程序，选择 属性。

2

在 常规 页底部找到 解除锁定 / Unblock。

3

勾选后点击 应用 → 确定。

4

重新双击打开安装包或应用。

常见问题

这些提示不代表应用一定有问题，更多是系统对未签名应用的默认拦截。

为什么会出现这些提示？

因为当前版本没有购买开发者证书进行代码签名，也没有完成 Apple notarization 或 Microsoft 信誉积累。系统无法确认开发者身份，所以会要求用户手动确认。

绕过后会不会每次都提示？

通常同一个应用版本只需要手动允许一次。更新新版本、换下载路径、重新解压或重新下载后，可能还会再次提示。

什么时候不应该继续打开？

如果安装包不是从作者指定渠道获取、文件大小明显不对、下载链接来自陌生人、或杀毒软件明确报毒，请不要绕过，先联系发布者确认。

后续正式版会解决吗？

正式对外分发会补齐代码签名。
